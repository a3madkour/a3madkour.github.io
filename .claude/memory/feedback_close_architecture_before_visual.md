When closing out one architectural topic and considering what to do next, do not pick the topic that gets us to a visual mockup faster. The user has twice (once on architecture v2 feedback, once on Research design) corrected me for moving to visual identity while content/architecture decisions were still open.

**Why:** They want the content shape settled before committing to visual identity, because layout decisions depend on what content structures exist. Skipping forward feels like prioritizing my own progress signal (a finished mockup) over the actual brief.

**How to apply:** Default to finishing every content/architecture thread before any typography, color, or homepage-mockup work. If multiple threads are open, ask the user which to close next rather than picking. When in doubt, choose the architecture topic over the visual one.
