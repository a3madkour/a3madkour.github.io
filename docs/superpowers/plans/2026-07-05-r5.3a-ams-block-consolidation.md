# R5.3a — AMS Block Consolidation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Collapse the 11 numbered AMS block shortcodes (identical modulo kind/class/tier/counter) into thin one-line wrappers over one shared `layouts/partials/ams-block.html`, with **byte-identical rendered output**. `proof.html` stays bespoke.

**Architecture:** One shared partial holds the render logic extracted verbatim from `theorem.html`, parameterized by a `dict` (`ctx`, `kind`, `class`, `tier`, `counter`). Each of the 11 numbered shortcodes becomes a single `{{- partial "ams-block.html" (dict …) -}}` call. Verification is a rendered-HTML diff (before vs. after must be empty) — server-rendered templates are deterministic, so no new tests are needed.

**Tech Stack:** Hugo templates (Go html/template), ox-hugo shortcode contract, existing `anchor-link.html` partial + `$page.Scratch` counters.

## Global Constraints

- **Byte-identical rendered output.** This is a pure refactor: the HTML Hugo emits for every block on every fixture page must be identical before and after (verified by `diff`). Whitespace deltas count — tune template trims (`{{-` / `-}}`) until the diff is empty.
- **`proof.html` is untouched** (the one non-numbered block — no counter, `<em>` header, `:of` arg, ∎ tombstone). Do not fold it into the shared partial.
- **Preserve the ref-block mechanism.** Each numbered block sets `$page.Scratch.Set "block-label-<id>" "<Kind> <N>"` (or `"<Kind> <N> (<Title>)"`) and passes the same string as the `anchor-link` aria label. `{{< ref-block "id" >}}` reads that Scratch entry. Reproduce exactly.
- **Preserve counter identity.** theorem/lemma/corollary/proposition share the `theorem-family` key; definition + the six soft blocks each keep their own `<kind>-counter` key.
- **Preserve `.block-header` structure/text** — `assets/js/block-renumber.js` rewrites its leading text on section-prefixed essays and must still match.
- No new shortcodes, kinds, tiers, CSS (§47), or JS. No change to `ref-block.html` or `block-renumber.js`.
- Spec: `docs/superpowers/specs/2026-07-05-r5.3a-ams-block-consolidation-design.md`.

- [ ] **Step 2: Create `layouts/partials/ams-block.html`**

This is `theorem.html`'s body verbatim, parameterized. Note `$.kind` (not `.kind`) inside the `with $id`/`with $title` blocks, since `.` is rebound there:

```go-html-template
{{- /* Shared AMS numbered-block renderer (R5.3a). Params:
       ctx     — the calling shortcode (.Page / .Get / .Inner)
       kind    — display label, e.g. "Theorem"
       class   — block class, e.g. "block-theorem"
       tier    — "block-strong" | "block-soft"
       counter — $page.Scratch counter key, e.g. "theorem-family"
       proof.html stays bespoke (unnumbered). Spec:
       docs/superpowers/specs/2026-07-05-r5.3a-ams-block-consolidation-design.md */ -}}
{{- $ctx := .ctx -}}
{{- $page := $ctx.Page -}}
{{- $n := add (int ($page.Scratch.Get .counter | default 0)) 1 -}}
{{- $page.Scratch.Set .counter $n -}}
{{- $title := $ctx.Get "title" -}}
{{- $id := $ctx.Get "id" -}}
<div class="{{ .class }} {{ .tier }}"{{ with $id }} id="{{ . }}"{{ end }}>
  <p class="block-header">
    {{ .kind }} {{ $n }}{{ with $title }} (<span class="block-title">{{ . }}</span>){{ end }}.
  </p>
  {{- with $id -}}
    {{- $aria := printf "%s %d" $.kind $n -}}
    {{- $page.Scratch.Set (printf "block-label-%s" $id) $aria -}}
    {{- with $title -}}{{ $aria = printf "%s %d (%s)" $.kind $n . -}}{{- end -}}
    {{ partial "anchor-link.html" (dict "id" $id "title" $aria) }}
  {{- end -}}
  <div class="block-body">{{ $ctx.Inner | markdownify }}</div>
</div>
```
