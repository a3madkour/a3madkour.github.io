# R6.1 CSS Spacing Scale Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace ~40 scattered raw rem spacing values in `assets/css/main.css` with a 12-step `--space-*` token scale, guarded by a new linter pair.

**Architecture:** Define the ladder once on `:root`; migrate rem-valued `gap`/`padding`/`margin` declarations to `var(--space-*)` by snapping the rare tail to nearest step; add a guard linter that fails on any un-tokenized in-scope rem ≥ 0.2rem (allowlisting sub-0.25rem micro-nudges). The linter is built first so its output is the migration worklist, then wired into CI only after migration completes green.

**Tech Stack:** Hand-rolled CSS processed by Hugo `resources.Get`; Python 3 stdlib linters with the `run(repo_root) -> (int, list[str])` + thin `main()` seam (R5.4); `tools/test_helpers.TempRepo` for linter tests; Playwright E2E (dev-only) + `check-contrast.py` + `ci-local.sh` as guards.

## Global Constraints

- **No Node/npm for the shipped site; Python tooling is stdlib-only.** The linter uses only the standard library.
- **Linter convention (R5.4):** every linter exposes `run(repo_root: Path) -> (int, list[str])` and a thin `main() -> int`; the paired test is `tools/test_check_<name>.py`.
- **Spacing tokens are mode-independent:** define the `--space-*` block ONCE on `:root`. Do NOT duplicate it in the `:root[data-theme="dark"]` block or the `@media (prefers-color-scheme: dark)` block (that duplication rule applies to color tokens only).
- **Option A is a deliberate visual change:** common values do not move; only the rare tail snaps. Verify visually via dev-server spot-checks at ~960px (author runs a tiling WM) and full width — the E2E suite is a functional regression guard, not a pixel verifier.
- **Do not run `hugo --minify` while a dev server is alive** (poisons dev CSS via MIME mismatch).
- **The canonical snap table** (raw rem → token) used by every migration task:

| Token | Value | Raw rem values that map to it |
|---|---|---|
| `--space-3xs` | 0.25rem | 0.2, 0.25, 0.3 |
| `--space-2xs` | 0.4rem | 0.35, 0.375, 0.4, 0.45 |
| `--space-xs` | 0.5rem | 0.5, 0.55 |
| `--space-sm` | 0.6rem | 0.6, 0.625 |
| `--space-md` | 0.75rem | 0.7, 0.75, 0.8, 0.85, 0.875, 0.9 |
| `--space-lg` | 1rem | 1, 1.1, 1.125 |
| `--space-xl` | 1.25rem | 1.2, 1.25 |
| `--space-2xl` | 1.5rem | 1.4, 1.5, 1.6 |
| `--space-3xl` | 2rem | 1.75, 1.8, 2 |
| `--space-4xl` | 2.5rem | 2.5 |
| `--space-5xl` | 3rem | 3 |

- **Allowlist (stay literal, never migrated):** `0.02, 0.05, 0.1, 0.125, 0.15` rem — sub-0.25rem micro-nudges. Note `0.2` and `0.3` are NOT allowlisted; they snap to `--space-3xs`.
- **Negative rem spacing:** map through calc — `-0.4rem` → `calc(-1 * var(--space-2xs))`, `-1.25rem` → `calc(-1 * var(--space-xl))`.

```python
#!/usr/bin/env python3
"""Spacing-token discipline linter (R6.1).

`assets/css/main.css` uses a 12-step `--space-*` scale for layout rhythm. This
linter fails if any in-scope spacing declaration (`gap`/`row-gap`/`column-gap`/
`padding*`/`margin*`) carries a bare rem literal instead of a `var(--space-*)`
reference, so magic-number spacing can't re-accrete.

Allowlisted sub-0.25rem micro-nudges (hairline border/alignment tweaks, not
rhythm) may remain literal. Note 0.2rem / 0.3rem are NOT allowlisted — they snap
to --space-3xs.

Stdlib only. Exits 0 when every in-scope rem is tokenized or allowlisted, 1
otherwise.
"""
from __future__ import annotations

import re
import sys
from pathlib import Path

# Sub-0.25rem micro-nudges that stay literal (see spec R6.1).
ALLOWLIST: set[float] = {0.02, 0.05, 0.1, 0.125, 0.15}

COMMENT_RE = re.compile(r"/\*.*?\*/", re.DOTALL)
# In-scope property at a declaration position: `<prop>:`
IN_SCOPE_PROP_RE = re.compile(
    r"(?<![\w-])"
    r"(gap|row-gap|column-gap|"
    r"padding(?:-(?:top|right|bottom|left|block|inline)(?:-(?:start|end))?)?|"
    r"margin(?:-(?:top|right|bottom|left|block|inline)(?:-(?:start|end))?)?)"
    r"\s*:\s*([^;{}]*)"
)
# A bare rem literal (optionally signed). `var(--space-…)` has no rem, so it
# never matches; `calc(-1 * var(--space-…))` likewise carries no bare rem.
REM_RE = re.compile(r"-?(\d*\.?\d+)rem\b")


def _allowlisted(num: str) -> bool:
    try:
        val = float(num)
    except ValueError:
        return False
    return any(abs(val - a) < 1e-6 for a in ALLOWLIST)


def find_violations(css: str) -> list[tuple[int, str, str]]:
    """Return (lineno, prop, raw_number) for every bare, non-allowlisted rem in
    an in-scope declaration. `raw_number` is the numeric text (no unit)."""
    css = COMMENT_RE.sub(lambda m: "\n" * m.group(0).count("\n"), css)
    violations: list[tuple[int, str, str]] = []
    for m in IN_SCOPE_PROP_RE.finditer(css):
        prop, value = m.group(1), m.group(2)
        for rem in REM_RE.finditer(value):
            num = rem.group(1)
            if _allowlisted(num):
                continue
            lineno = css.count("\n", 0, m.start()) + 1
            violations.append((lineno, prop, num))
    return violations


def run(repo_root: Path) -> tuple[int, list[str]]:
    css_path = repo_root / "assets" / "css" / "main.css"
    if not css_path.exists():
        return (1, [f"main.css not found at {css_path}"])
    css = css_path.read_text(encoding="utf-8")
    errors = [
        f"{css_path}:{ln}: raw spacing '{num}rem' in `{prop}` — "
        f"use var(--space-*) or allowlist a micro-nudge"
        for ln, prop, num in find_violations(css)
    ]
    return (1 if errors else 0, errors)


def main() -> int:
    repo_root = Path(__file__).resolve().parent.parent
    rc, errors = run(repo_root)
    if errors:
        for e in errors:
            print(f"error: {e}", file=sys.stderr)
        print(f"\n{len(errors)} un-tokenized spacing value(s).", file=sys.stderr)
        return rc
    print("OK — every in-scope spacing value is tokenized or allowlisted.")
    return 0


if __name__ == "__main__":
    sys.exit(main())
```
