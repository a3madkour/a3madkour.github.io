# R5.3b — graph-panel single-partial Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Collapse the three near-identical `layouts/partials/{garden,research,works}/graph-panel.html` partials into one shared `layouts/partials/graph-panel.html` + three thin wrappers, with the single intended change being research/works gaining `aria-label="Graph filters"` on their toolbar (standardize up).

**Architecture:** One shared partial parameterized by a `dict` (`id`, `title`, `ariaLabel`, `section`). The three section partials become one-line wrappers passing their constants. The 9 include sites stay unchanged. Verification is a rendered-HTML diff that must show **only** the two toolbar-aria-label additions.

**Tech Stack:** Hugo partials; the already-shared `graph-legend.html`; graph-core adapters that look up the panel by id.

## Global Constraints

- **Intended semantic delta:** research + works panel markup gains `aria-label="Graph filters"` on the `.graph-toolbar--panel` div (standardize up). Garden markup is byte-identical (minified). Research/works also get a **cosmetic `<aside>` attribute reorder** (their source order differed from the shared partial's) — semantically identical HTML, verified inert in Step 4. No content/structural change beyond these.
- **Preserve per-section panel `id`** (`<section>-graph-panel`): graph-core adapters do `document.getElementById(...)` and the close button's `aria-controls` must match.
- **Preserve `aria-hidden="true" inert` initial (closed) state**, the empty `.graph-toolbar--panel` + empty `.graph-panel-canvas` (JS-populated), and the `graph-legend.html` call with the section's own `section` + `variant "panel"`.
- **9 call sites unchanged**: `garden/{single,list}.html`, `research/list.html`, `research-question/single.html`, `research-theme/single.html`, `works/list.html`, `works-{games,music,poetry}/single.html`.
- Spec: `docs/superpowers/specs/2026-07-05-r5.3b-graph-panel-single-partial-design.md`.

```go-html-template
{{- /* Shared graph-panel skeleton (R5.3b). The ONLY place a graph side-panel is
```

- [ ] **Step 4: Rebuild (minified) and verify — the parity gate**

```bash
rm -rf public && HUGO_ENVIRONMENT=production hugo --gc --minify
diff -rq /tmp/r53b-before/garden public/garden && echo "GARDEN byte-identical ✓"
diff -rq /tmp/r53b-before/research public/research   # will list files (expected — see below)
diff -rq /tmp/r53b-before/works public/works         # will list files (expected)
```

Then confirm the research/works deltas are **localized to the panel and semantically inert** with this check:
```bash
python3 - <<'PY'
import re
def check(before, after, label):
    a = open(before, encoding='utf-8').read(); b = open(after, encoding='utf-8').read()
    def split(html):
        m = re.search(r'<aside [^>]*graph-panel[^>]*>.*?</aside>', html, re.S)
        return html[:m.start()], m.group(0), html[m.end():]
    pa, asa, poa = split(a); pb, asb, pob = split(b)
    print(label, "pre-aside identical:", pa==pb, "| post-aside identical:", poa==pob)
    for tok in ['id='+label+'-graph-panel','aria-hidden=true','inert','role=region',
                'graph-panel-canvas','graph-legend','graph-panel-close','aria-controls']:
        assert (tok in asa)==(tok in asb), (label, tok)
    assert 'aria-label=Graph filters' in asb.replace('"',''), (label,'toolbar aria missing')
    print("   all panel semantic tokens preserved + toolbar aria added ✓")
check("/tmp/r53b-before/research/index.html","public/research/index.html","research")
check("/tmp/r53b-before/works/index.html","public/works/index.html","works")
PY
```
Expected:
- **garden**: `GARDEN byte-identical ✓` (empty diff). This proves the shared partial reproduces a panel exactly.
- **research + works**: `pre-aside identical: True | post-aside identical: True` for both, and `all panel semantic tokens preserved + toolbar aria added ✓`. The two differences vs. before are (1) the `<aside>` opening tag's attributes reorder to the partial's canonical order `class, id, role, aria-label, aria-hidden, inert` — **semantically identical HTML** (attribute order is irrelevant to browsers/CSS/JS), and (2) the toolbar div gains `aria-label="Graph filters"` (the intended standardize-up).

If garden is not byte-identical, or the python check fails (a token dropped, or content outside the aside changed), that is a real regression — fix before committing. Do NOT edit call sites or fixtures to force a match.
