# R5.3a — AMS block consolidation (design)

**Date:** 2026-07-05
**Status:** Design approved; ready for `superpowers:writing-plans`.
**Parent:** row R5.3 in [`2026-07-03-audit-remediation-roadmap.md`](2026-07-03-audit-remediation-roadmap.md). R5.3 was split (user decision): **R5.3a = AMS blocks (this doc)**; R5.3b = graph-panel single-partial, a separate later cycle.

## Goal

Collapse the 12 AMS-style block shortcodes (214 LOC, `layouts/shortcodes/{theorem,lemma,corollary,proposition,definition,remark,example,note,claim,conjecture,axiom,proof}.html`) — 11 of which are identical modulo (kind label, CSS class, tier, counter key) — into thin one-line wrappers over a single shared partial, with **zero rendered-output change**.

## Decision (brainstorm)

**11 shared + proof bespoke.** One shared partial renders the 11 numbered blocks; `proof.html` stays as-is (it is the outlier: no counter, `<em>` header, `:of` arg instead of `:title`, ∎ tombstone — and it isn't duplicated anyway, so nothing is lost by leaving it). This keeps the shared partial free of proof-only conditionals.

## The three block shapes (current)

| Shape | Blocks | counter key | tier class |
|---|---|---|---|
| theorem-family | theorem, lemma, corollary, proposition | `theorem-family` (shared) | `block-strong` |
| independent-strong | definition | `definition-counter` | `block-strong` |
| independent-soft | remark, example, note, claim, conjecture, axiom | `<kind>-counter` | `block-soft` |
| proof (bespoke) | proof | — (none) | — (`block-proof`) |

The 11 numbered blocks differ **only** in: `kind` display label, `class` (`block-<kind>`), `tier` (`block-strong`/`block-soft`), and `counter` key. Their header format (`Kind N (Title).`), wrapper, anchor-link emission, and body are byte-identical.

## Architecture

### New partial: `layouts/partials/ams-block.html`

Render logic extracted verbatim from `theorem.html`. Signature — called as `{{ partial "ams-block.html" (dict …) }}` with:

- `ctx` — the calling shortcode (for `.Page`, `.Get "title"`, `.Get "id"`, `.Inner`).
- `kind` — display label, e.g. `"Theorem"`.
- `class` — block class, e.g. `"block-theorem"`.
- `tier` — `"block-strong"` or `"block-soft"`.
- `counter` — the `Scratch` counter key, e.g. `"theorem-family"`.

Behavior (exactly today's `theorem.html`, parameterized):

```go-html-template
{{- $ctx := .ctx -}}
{{- $page := $ctx.Page -}}
{{- $n := add (int ($page.Scratch.Get .counter | default 0)) 1 -}}
{{- $page.Scratch.Set .counter $n -}}
{{- $title := $ctx.Get "title" -}}
{{- $id := $ctx.Get "id" -}}
<div class="{{ .class }} {{ .tier }}"{{ with $id }} id="{{ . }}"{{ end }}>
  <p class="block-header">
    {{ .kind }} {{ $n }}{{ with $title }} (<span class="block-title">{{ . }}</span>){{ end }}.
  </p>
  {{- with $id -}}
    {{- $aria := printf "%s %d" $.kind $n -}}
    {{- $page.Scratch.Set (printf "block-label-%s" $id) $aria -}}
    {{- with $title -}}{{ $aria = printf "%s %d (%s)" $.kind $n . -}}{{- end -}}
    {{ partial "anchor-link.html" (dict "id" $id "title" $aria) }}
  {{- end -}}
  <div class="block-body">{{ $ctx.Inner | markdownify }}</div>
</div>
```

### The 11 wrappers

Each numbered shortcode becomes one `partial` call, e.g.:

```
layouts/shortcodes/theorem.html:
{{ partial "ams-block.html" (dict "ctx" . "kind" "Theorem" "class" "block-theorem" "tier" "block-strong" "counter" "theorem-family") }}

layouts/shortcodes/remark.html:
{{ partial "ams-block.html" (dict "ctx" . "kind" "Remark" "class" "block-remark" "tier" "block-soft" "counter" "remark-counter") }}
```

Full per-block param table (kind / class / tier / counter) is in the plan.

### `proof.html` — unchanged

Left exactly as-is (bespoke render).

## Critical invariants (must not regress)

1. **ref-block cross-references.** Each numbered block sets `Scratch "block-label-<id>" = "Kind N"` (or `"Kind N (Title)"`) and passes the same string as the `anchor-link` aria label. `{{< ref-block "id" >}}` reads that Scratch entry. The shared partial reproduces this exactly (note `$.kind`/`$.counter` inside the `with` block, since `.` is rebound). Exercised by `content/essays/example-blocks-crossref/`.
2. **Counter identity.** theorem-family blocks must keep sharing the `theorem-family` key (one counter across theorem/lemma/corollary/proposition); each independent block keeps its own `<kind>-counter` key. Exercised by numbering in `example-long-numbering` + `example-five`.
3. **Section-prefixed renumbering.** `assets/js/block-renumber.js` rewrites `.block-header` leading text on section-prefixed essays; the partial must keep emitting the identical `.block-header` structure/text so the JS still matches. Exercised by `example-long-numbering` (`block_numbering: "section-prefixed"`).
4. **Whitespace/markup byte-identity.** The rendered HTML must be byte-for-byte identical to today (verified by diff), so no downstream CSS/JS/anchor selector shifts.

## Verification

Pure Hugo templates → deterministic output, so the parity proof is a **rendered-HTML diff**:

1. Build the block fixtures on the current code (`git stash` or a scratch checkout), capture the rendered `.block-*` HTML from `example-blocks-crossref`, `example-long-numbering`, `example-five`.
2. Apply the refactor, rebuild, diff. **Expected: empty diff** (byte-identical). Any diff = a regression to fix before commit.
3. `python3 tools/check_anchor_link.py` + its unit test stay green (the block anchor affordance).
4. Full `tools/ci-local.sh` green (fixtures, smoke, css-refs, anchor-link, E2E — the whole gate).

No new tests: the rendered diff is a complete behavior-preservation proof for server-rendered markup, and the existing linters guard the anchor + fixture contracts.

## Definition of done

`ams-block.html` created; the 11 numbered shortcodes reduced to one-line wrappers; `proof.html` untouched; rendered-HTML diff empty on all block fixtures; `check_anchor_link` + `ci-local.sh` green; 214 LOC → ~11 wrappers + `proof.html` + one partial. CLAUDE.md's "Semantic blocks" section updated to note the shared partial. Then R5.3b (graph-panel single-partial) or R5.4 (Python tooling dedup).

## Non-goals (YAGNI)

- No change to `proof.html`, `ref-block.html`, `block-renumber.js`, or CSS §47.
- No folding proof into the shared partial (explicitly rejected in brainstorm).
- No change to block kinds, numbering rules, tiers, or the ox-hugo `#+attr_shortcode` contract.
- Graph-panel consolidation is R5.3b (separate cycle).
