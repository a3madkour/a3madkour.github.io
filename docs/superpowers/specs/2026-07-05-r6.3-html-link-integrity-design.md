# R6.3 — Built-HTML link-integrity crawl (design)

**Date:** 2026-07-05
**Roadmap row:** R6.3 (`docs/superpowers/specs/2026-07-03-audit-remediation-roadmap.md`, Tier R6, Severity Low, trigger-gated).
**Status:** designed. Opened deliberately (the author chose to build the
trigger fixtures now rather than wait for real content — see
[[feedback_trigger_gated_make_fixture.md]]).

## Problem

The existing `check_*_links.py` linters validate **frontmatter-declared** refs
*pre-build* (garden/research/works/streams/library round-trips). Nothing crawls
the **rendered** `public/` for broken `<a href>`s — a heading rename that
orphans a `#fragment`, a mistyped `/section/slug/`, or a link to a draft page
would ship undetected.

R6.3 was deferred as trigger-gated ("adequate for a fixture-only site"). Two
findings reopen it now:

1. The built site is **not** link-sparse: **~3,416 `<a href>` across 123 HTML
   pages** (nav/footer chrome repeated site-wide + content links). There is real
   integrity to guard today.
2. Hugo's HTML minifier **strips attribute quotes** (`href=/essays/`, not
   `href="/essays/"`), so a naive regex crawler would silently miss most links.
   A real HTML parser is mandatory.

## Goal (Option A — full link integrity, offline)

A built-HTML crawler that verifies every **internal** `<a href>` resolves to a
file in `public/`, **and** every fragment resolves to a real anchor in the
target document. External links are out of scope (no network in CI — the
Google-Fonts flake lesson, [[reference_lhci_google_fonts_flakiness]]). Plus a
few augmented fixtures that exercise the cross-page + anchor paths that barely
exist today.

## Component 1 — the crawler `tools/check_html_links.py` (33rd linter pair)

Stdlib only. `run(public: Path) -> tuple[int, list[str]]` + thin `main()`,
following the R5.4 seam. Like `check_anchor_link` / `check_smoke`, it operates on
the **built `public/`** (a "public-path" seam outlier, not `repo_root`), so it
runs in CI **after** the Hugo build.

**Pass 1 — index.** Walk `public/**/*.html`. Parse each with a stdlib
`html.parser.HTMLParser` subclass. For each document record:
- `anchors[relpath]`: the set of every `id="…"` and legacy `name="…"` value.
- `links[relpath]`: the list of `(href, is_in_a_tag)` — only `<a href>` values.

Parsing with `HTMLParser` (not regex) is required because minified attributes are
unquoted; `HTMLParser.handle_starttag` yields a clean `attrs` dict regardless of
quoting.

**Pass 2 — resolve.** For each internal `<a href>` (see rules), resolve to a
`public/` file and, for HTML targets with a fragment, to an anchor in that file.
Collect a sorted list of human-readable errors: `SOURCE.html: <a href="X"> →
<reason>` (missing target file / missing anchor `#id` in TARGET).

Returns `(1, errors)` if any, else `(0, [])`.

### Internal vs skipped

- **Internal (checked):**
  - root-relative: `/foo/`, `/foo/#bar`
  - same-domain absolute: `https://a3madkour.github.io/foo/` → strip the site
    `baseURL` origin, treat as `/foo/`
  - pure fragment: `#bar` → resolve against the current document
  - relative: `foo/`, `../bar/` → resolve against the current page's URL path
- **Skipped (not link-integrity in scope):**
  - other-domain `http(s)://…`
  - `mailto:`, `tel:`, `data:`, `javascript:`
  - protocol-relative `//cdn…` (external)

### File resolution

1. Strip any `?query` and split off `#fragment`.
2. Map the path to a file under `public/`:
   - `/` → `public/index.html`
   - `/foo/` → `public/foo/index.html`
   - `/foo` → `public/foo/index.html` if it exists, else `public/foo.html`
   - `/foo.ext` (any non-`/`-terminated path with an extension, e.g.
     `/essays/index.xml`, `/library/covers/x.jpg`) → `public/foo.ext`
3. If the resolved file does not exist → error (broken target). This correctly
   catches links to **draft** pages (absent from a production build).

### Fragment resolution (HTML targets only)

- Empty fragment (`#`) and `#top` → always valid (browser top-scroll; no element
  required).
- Any other `#name` → must be in the target document's `anchors` set (`id` or
  `name`). Missing → error.
- Non-HTML targets (`.xml`, `.jpg`, …) → file-existence only; fragments on them
  are ignored (not applicable).

### Performance

123 pages / ~3.4k links: each page parsed once into `anchors`/`links`, then a
pure in-memory resolve. Trivially fast; no ordering constraints.

## Component 2 — fixtures (augment existing)

Add cross-links into existing fixture bodies (obviously-dummy prose only, per the
fixture-content rule) so the crawler's richer paths are exercised by live,
**valid** content:

- At least one **cross-page anchor** link: an essay body → another page's real
  generated heading slug (e.g. `/essays/<slug>/#the-classic-trade-off`). If that
  heading is later renamed, its slug changes and the crawler flags the orphaned
  link — the headline value of the whole slice.
- At least one **cross-section** link: an essay → a garden note or research
  question (e.g. `/garden/bayesian-statistics/`), or garden → research.
- Keep it to ~3 augmented fixtures; do not add pages that exist only to be
  crawled (that is the rejected "new dedicated pages" approach).

All augmentations must resolve (green). The target heading slugs are Hugo's
auto-generated github-style slugs; the fixture author verifies each against the
built HTML before committing.

**Broken cases are NOT authored as live fixtures** (they would fail CI). They are
exercised in the linter's unit tests with synthetic HTML: missing target file,
missing `#anchor`, a renamed-heading orphan, external link skipped, `mailto:`
skipped, `#`/`#top` accepted, unquoted-attribute parsing, relative-path
resolution.

## Pre-existing breakage

The crawler's first run against the current built site may surface **real
pre-existing broken links**. That is the feature working; **fixing any it finds
is in scope for this slice** (the linter cannot be wired green otherwise).

## Verification

1. `python3 tools/check_html_links.py` → exits 0 on a clean production build
   (`rm -rf public resources/_gen && hugo --gc --minify --quiet`) — after fixing
   any real breakage it surfaces. (Clean-build first — a stale dev-server
   `public/` with draft pages false-positives, see
   [[reference_dev_server_stale_draft_page_weights]].)
2. `python3 -m unittest tools/test_check_html_links.py` → all pass (synthetic-HTML
   cases above).
3. `tools/ci-local.sh` green — full linter suite incl. the new pair; Playwright
   E2E **9/9** (fixture augmentations are prose links, no runtime change); build.
4. `python3 tools/check-contrast.py` green (unaffected; part of the gate).

## Out of scope / deferred

- External link checking (network HEAD) — flaky/slow in CI; rejected.
- Non-`<a>` resources (`<img src>`, `<script src>`, `<link href>`) — R6.3 is the
  `<a href>` reading-flow crawl; asset integrity is a separate concern (and
  `check_page_weights` / the build already exercise asset presence).
- New dedicated fixture pages (augment-existing chosen instead).

## Naming note

`check_html_links.py` — deliberately distinct from the frontmatter
`check_garden_links.py` / `check_research_links.py` / etc., which validate
declared refs pre-build. This one crawls rendered HTML.
