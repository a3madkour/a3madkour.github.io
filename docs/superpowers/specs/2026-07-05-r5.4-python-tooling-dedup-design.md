# R5.4 — Python tooling dedup (design)

**Date:** 2026-07-05
**Status:** Design approved; ready for `superpowers:writing-plans`.
**Parent:** row R5.4 in [`2026-07-03-audit-remediation-roadmap.md`](2026-07-03-audit-remediation-roadmap.md). The final Tier-R5 dedup (closes the "copy instead of abstract" story on the Python side). Scope: **all four** sub-items (user decision).

## Goal

Remove the four duplication classes the audit flagged in `tools/`, with **zero change to any linter's pass/fail behavior** (the 30 linter+test pairs are the guard):
1. 22 test files re-roll identical tempdir scaffolding → one shared `test_helpers.py`.
2. `citations.yaml` has two divergent parsers → one canonical.
3. 6 linters roll their own frontmatter/YAML parsing → the shared parser (where the shape matches).
4. Inconsistent entrypoints (15/30 lack a `run()` seam; return arity varies) → standardize on `run(repo_root) -> (int, list[str])` + thin `main()`.

## Key decisions (brainstorm)

- **Keep `check_fixtures.py` as the shared-parser home** (it already exports `parse_scalar` + frontmatter parsing, imported by ~15 linters). No new `lint_common.py` — follows the established pattern, minimizes churn. (Importing `check_fixtures` runs only its module-level defs, not `main()` — safe, and already the norm.)
- **Migrate only genuinely-same-shape YAML rollers** in item 3; linters parsing a different structure (e.g. `check_icon_attribution`, `check_library_shelves` parse non-frontmatter shapes) stay as-is, with a one-line comment noting why.
- **Tests importing a shared test helper is clean** (no linter-imports-linter concern) — `test_helpers.py` is test-only.

## Architecture

### §1 — `tools/test_helpers.py` (item 1)

New test-only module:
```python
class TempRepo:
    """Throwaway repo root for linter tests. Replaces the per-file TempRepo
    that ~22 test_check_*.py files each re-roll."""
    def __init__(self) -> None: self.root = Path(tempfile.mkdtemp())
    def write(self, rel: str, text: str) -> Path:
        """Write text to <root>/<rel>, creating parent dirs. Returns the path."""
        p = self.root / rel; p.parent.mkdir(parents=True, exist_ok=True)
        p.write_text(text, encoding="utf-8"); return p
    def cleanup(self) -> None: shutil.rmtree(self.root, ignore_errors=True)
```
Each test file: `from test_helpers import TempRepo`, delete its local copy, keep its domain writers as thin wrappers (e.g. `def write_note(self, slug, body): return self.repo.write(f"content/garden/{slug}/index.md", body)`). setUp/tearDown stay per-file (they're one line each) or use a shared `LinterTestCase` mixin if the plan finds it worth it. The plan enumerates the exact test files that carry a `TempRepo` (not all 30 do).

### §2 — parser consolidation (items 2 + 3)

- **Citation parser:** the richer `check_citations.parse_citations_yaml` (→ dict of entries) is canonical. `check_fixtures.parse_citations_yaml` (→ set of keys) becomes `return set(check_citations.parse_citations_yaml(text_or_path))`. **The plan must first confirm no import cycle** (`check_citations` must not import `check_fixtures`, or the canonical parser moves to `check_fixtures` instead). One parser, one behavior.
- **YAML rollers:** the linters that hand-roll frontmatter parsing of the *same* `---`-delimited shape import `check_fixtures`'s parser instead. The plan enumerates which of the ~6 candidates genuinely match (vs. parse a different structure and stay).

### §3 — entrypoint seam standardization (item 4)

Every linter exposes:
```python
def run(repo_root: Path) -> tuple[int, list[str]]:
    """Return (exit_code, errors). The importable seam for tests + main()."""
    ...
    return (1 if errors else 0, errors)

def main() -> int:
    rc, errors = run(Path(__file__).resolve().parent.parent)
    for e in errors: print(...)
    return rc

if __name__ == "__main__":
    sys.exit(main())
```
The ~15 laggards (main-only, or inline-`__main__` with no `main()`) get this seam. Any `(rc, errors, warnings)` arity collapses to `(rc, errors)` — warnings, where they exist, either fold into a separate return path the plan specifies or print inside `run`/`main` without changing the `(rc, errors)` contract. **CI is unaffected**: it calls `python3 tools/check_X.py` → `main()`, whose external behavior (stdout + exit code) is unchanged.

## Invariants (must not regress)

1. **No linter's pass/fail output changes.** Same errors, same exit codes, same stdout messages on the same inputs. The paired unit tests assert this — every one must stay green.
2. **CI invocation unchanged** — `python3 tools/check_X.py` + `python3 -m unittest tools/test_check_X.py` both still work for all 30 pairs.
3. **The R1.1-hardened `parse_scalar`** stays the one true frontmatter scalar parser — do not fork or weaken it.
4. **Sibling-less linters** (`check_smoke`, `check_graph_chrome`) get the `run()` seam too (consistency) but have no paired test — verify via `ci-local`.

## Verification

The 30 linter+test pairs are the behavior guard (pure refactor). Staged:
1. **`test_helpers.py` + migrate tests** → `python3 -m unittest` all migrated `test_check_*.py` green.
2. **Parser consolidation** → the affected linters + their tests green (`check_citations`, `check_fixtures`, the migrated rollers).
3. **run-seam standardization** → every linter still runs standalone (`python3 tools/check_X.py`) and every paired test green.
4. **Full `tools/ci-local.sh`** green (all linters + tests + build + E2E).

No new product tests: the existing pairs already pin each linter's behavior; the refactor must keep them green. Add a couple of `test_helpers.py` self-tests only if the plan finds `TempRepo` warrants them (it's ~10 lines — likely covered transitively by every migrated test).

## Definition of done

`test_helpers.py` created + the ~22 TempRepo-carrying test files migrated; one canonical `citations.yaml` parser; same-shape YAML rollers on the shared parser; every linter on the `run(repo_root) -> (int, list[str])` seam; all 30 pairs + `ci-local.sh` green; CLAUDE.md's tooling description updated (shared test scaffold + `run()` seam convention). Closes the audit's R5.4 row and the "copy instead of abstract" story. Then R6 (optional).

## Non-goals (YAGNI)

- No new linter behavior, no new checks, no fixture changes.
- No `lint_common.py` (check_fixtures stays the shared home).
- No migration of genuinely-different-shape parsers (documented case-by-case).
- No change to CI workflow invocations or `ci-local.sh` ordering (beyond staying green).
