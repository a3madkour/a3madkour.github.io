# R5.3b — graph-panel single-partial (design)

**Date:** 2026-07-05
**Status:** Design approved; ready for `superpowers:writing-plans`.
**Parent:** row R5.3 in [`2026-07-03-audit-remediation-roadmap.md`](2026-07-03-audit-remediation-roadmap.md). R5.3 was split: R5.3a (AMS blocks) shipped `3925b6c`; **R5.3b = graph-panel single-partial (this doc)**. Also closes the structural remainder of R3.2 (graph-panel skeleton reconciliation).

## Goal

Collapse the three near-identical graph-panel partials (`layouts/partials/{garden,research,works}/graph-panel.html`) into one shared `layouts/partials/graph-panel.html` + three thin wrappers. After R5.2 normalized works (aria-hidden+inert, empty toolbar, `<div>` canvas), the three differ **only** in id / title / aria-label / legend-section.

## Current divergence (verified)

All three are the same skeleton — `<aside class="graph-panel" … aria-hidden="true" … inert>` → header (title + `.graph-panel-close`) → `.graph-toolbar--panel` → `.graph-panel-canvas` → `graph-legend.html` — differing only in:

| | garden | research | works |
|---|---|---|---|
| panel `id` | `garden-graph-panel` | `research-graph-panel` | `works-graph-panel` |
| `.graph-panel-title` | `Graph` | `Research graph` | `Constellation` |
| aside `aria-label` | `Garden graph` | `Research graph` | `Works constellation` |
| legend `section` | `garden` | `research` | `works` |

**One pre-existing drift:** garden's toolbar div is `<div class="graph-toolbar graph-toolbar--panel" aria-label="Graph filters">` (with a comment); research/works are bare `<div class="graph-toolbar graph-toolbar--panel"></div>`.

## Decision

**Thin wrappers, call sites unchanged** (mirrors R5.3a + minimizes churn/risk). The three `{garden,research,works}/graph-panel.html` files become one-line wrappers over the shared partial; the **9 include sites stay unchanged** (they still call `garden/graph-panel.html` etc.).

**Standardize the toolbar `aria-label` UP** (brainstorm decision): the shared partial always emits `aria-label="Graph filters"` on the toolbar div, so garden keeps its affordance and research/works gain it. This is the single intended rendered delta — a minor a11y win, and precisely the "reconcile to one skeleton" R3.2 flagged.

## Architecture

### New shared partial: `layouts/partials/graph-panel.html`

Params (dict): `id`, `title`, `ariaLabel`, `section`. Renders:

```go-html-template
{{- /* Shared graph-panel skeleton (R5.3b). The ONLY place a graph side-panel
       is constructed. Params: dict "id" "title" "ariaLabel" "section".
       Toolbar + canvas are JS-populated by the section's graph-core adapter;
       the legend is server-rendered by graph-legend.html. */ -}}
<aside class="graph-panel" id="{{ .id }}" role="region" aria-label="{{ .ariaLabel }}" aria-hidden="true" inert>
  <header class="graph-panel-header">
    <span class="graph-panel-title">{{ .title }}</span>
    <button type="button" class="graph-panel-close" aria-controls="{{ .id }}" aria-label="Close graph panel">×</button>
  </header>
  <div class="graph-toolbar graph-toolbar--panel" aria-label="Graph filters"></div>
  <div class="graph-panel-canvas"></div>
  {{ partial "graph-legend.html" (dict "section" .section "variant" "panel") }}
</aside>
```

### The three wrappers

```
layouts/partials/garden/graph-panel.html:
{{ partial "graph-panel.html" (dict "id" "garden-graph-panel" "title" "Graph" "ariaLabel" "Garden graph" "section" "garden") }}

layouts/partials/research/graph-panel.html:
{{ partial "graph-panel.html" (dict "id" "research-graph-panel" "title" "Research graph" "ariaLabel" "Research graph" "section" "research") }}

layouts/partials/works/graph-panel.html:
{{ partial "graph-panel.html" (dict "id" "works-graph-panel" "title" "Constellation" "ariaLabel" "Works constellation" "section" "works") }}
```

### Call sites — unchanged

The 9 includes stay as-is: `garden/{single,list}.html`, `research/list.html`, `research-question/single.html`, `research-theme/single.html`, `works/list.html`, `works-{games,music,poetry}/single.html`.

## Invariants (must not regress)

1. **Panel id per section** — the graph-core adapters look up the panel by `document.getElementById("<section>-graph-panel")`, and the close button's `aria-controls` must match. Preserve exactly.
2. **`aria-hidden="true" inert` initial state** — graph-core's `openPanel`/`closePanel` toggle these; the panel must start closed.
3. **Empty `.graph-toolbar--panel` + empty `.graph-panel-canvas`** — JS-populated at runtime; must stay empty in markup.
4. **`graph-legend.html` called with the section's own `section` + `variant "panel"`** — legend swatch colors depend on it.
5. **Note:** no partial takes the page context (`.`) — the shared partial is pure-param, so passing the dict (not `.`) is correct; `graph-legend.html` inside it also gets a fresh dict.

## Verification

Verify against the **minified** build (what ships), not raw output. (Raw HTML preserves each source partial's `<aside>` line-breaking + attribute order, so unifying three differently-formatted sources into one partial changes *raw* formatting cosmetically; the minifier erases that. The minifier does still preserve *attribute order*, so research/works — whose source asides were `id`-first — show a cosmetic aside attribute-reorder to the partial's canonical order. That is semantically identical HTML.)

1. **Garden minified byte-identical** — proves the shared partial reproduces a panel exactly (garden's source aside order already matches the partial's).
2. **Research/works: change localized to the panel + semantically inert.** Everything outside the `<aside>` byte-identical; inside, all semantic tokens preserved (id, `aria-hidden`, `inert`, `role=region`, canvas, legend, close button, `aria-controls`); the only deltas are (a) the cosmetic aside attribute-reorder and (b) the intended toolbar `aria-label="Graph filters"`. A small `python3` check (in the plan) asserts pre/post-aside identity + token preservation.
3. **E2E graph specs green** (`npx playwright test`) — the works-graph spec exercises the panel-open path; garden/research mount specs unaffected.
4. **Full `tools/ci-local.sh` green.**

No new tests: the minified-diff analysis + existing E2E (panel open) + linters guard it.
