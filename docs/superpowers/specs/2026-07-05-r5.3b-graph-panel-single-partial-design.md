## Goal

Collapse the three near-identical graph-panel partials (`layouts/partials/{garden,research,works}/graph-panel.html`) into one shared `layouts/partials/graph-panel.html` + three thin wrappers. After R5.2 normalized works (aria-hidden+inert, empty toolbar, `<div>` canvas), the three differ **only** in id / title / aria-label / legend-section.

## Decision

**Thin wrappers, call sites unchanged** (mirrors R5.3a + minimizes churn/risk). The three `{garden,research,works}/graph-panel.html` files become one-line wrappers over the shared partial; the **9 include sites stay unchanged** (they still call `garden/graph-panel.html` etc.).

**Standardize the toolbar `aria-label` UP** (brainstorm decision): the shared partial always emits `aria-label="Graph filters"` on the toolbar div, so garden keeps its affordance and research/works gain it. This is the single intended rendered delta â€” a minor a11y win, and precisely the "reconcile to one skeleton" R3.2 flagged.

## Invariants (must not regress)

1. **Panel id per section** â€” the graph-core adapters look up the panel by `document.getElementById("<section>-graph-panel")`, and the close button's `aria-controls` must match. Preserve exactly.
2. **`aria-hidden="true" inert` initial state** â€” graph-core's `openPanel`/`closePanel` toggle these; the panel must start closed.
3. **Empty `.graph-toolbar--panel` + empty `.graph-panel-canvas`** â€” JS-populated at runtime; must stay empty in markup.
4. **`graph-legend.html` called with the section's own `section` + `variant "panel"`** â€” legend swatch colors depend on it.
5. **Note:** no partial takes the page context (`.`) â€” the shared partial is pure-param, so passing the dict (not `.`) is correct; `graph-legend.html` inside it also gets a fresh dict.
