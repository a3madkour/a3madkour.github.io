**Status:** Scoped + decision locked; ready to implement next session. This is a *brief* (scope + decisions), not a task-by-task plan — draft the plan via `superpowers:writing-plans` when implementation begins, per [[feedback-design-batch-no-plan-until-implement]].

## Why R5.1 comes first

The audit's second systemic story: **the test pyramid has no top half** — 30 Python linter pairs validate static markup shape, but *zero* automated coverage of runtime behavior (theme cycle, filter chips, 3 graph runtimes, search modal, cite/poetry/explorables, anchor-link, block-renumber). Every runtime bug in CLAUDE.md's gotcha log shipped and was caught by eyeball. **R5.1 is the prerequisite for the rest of Tier R5** — R5.2 (graph-core extraction), R5.3 (AMS-block + graph-panel consolidation, which subsumes R3.2's deferred structural piece), and R5.4 (Python tooling dedup) are all refactors that must be guarded against behavioral drift by this harness. Land it first; then every subsequent extraction is safe.

## Decision (locked 2026-07-04)

**Playwright, Node, dev-only.** Real-browser E2E over the built `public/` site. Chosen over playwright-python and jsdom for full real-runtime fidelity (real layout, focus, ARIA, d3 graph mount).

**The no-npm boundary — explicit:** CLAUDE.md's "No npm. Python tooling is stdlib-only" is about the **shipped site** and its **content pipeline**. This harness is **dev/CI tooling only** — it does not ship, is not part of the Hugo build, and adds nothing to any page's bytes. That distinction is the whole basis for the choice.

- `package.json` with **devDependencies only** (`@playwright/test`); `node_modules/` already gitignored.
- Tests under `tests/e2e/*.spec.ts` drive real Chromium over `./public`.
- CI: `npx playwright test` **after** the Hugo build (+ Pagefind index, so search works).

## First tests — a *thin smoke suite* (breadth over depth)

Start with one high-value assertion per major runtime surface. Depth comes later, per-module, as R5.2–R5.4 touch each.

1. **Theme cycle** — click the toggle: system → light → dark → system; assert `data-theme` on `<html>` + `localStorage['theme-pref']`.
2. **Filter chips** (`/essays/` or `/garden/`) — click a tag chip; assert the visible tile count drops and non-matching tiles get `[hidden]`; "All" clears.
3. **Search modal** — `/` opens the `<dialog>`; type a query; assert results render with `role="option"` + the input's `aria-activedescendant` updates on ArrowDown (guards R3.3).
4. **Cite modal** — click a `[Author Year]` cite; `<dialog>` opens; ArrowRight moves tabs (guards R3.4); copy button writes clipboard.
5. **Graph mount** (`/garden/graph/` or the panel) — assert the SVG gets nodes after d3 force init (this is the R5.2 safety net — the three graph runtimes are what R5.2 dedupes).
6. **no-js** — assert `<html>` loses `no-js` after load (guards R3.6).

Keep each test resilient (wait for elements, not fixed timeouts). The suite should run in well under a minute.

## Wiring notes

- **Serving `public/`:** use Playwright's `webServer` config (e.g. `python3 -m http.server` against `public/`, or `npx serve public`) so `<dialog>`, module scripts, and Pagefind fetch work over http:// (not file://). Pagefind needs the index built first.
- **CI:** new job or steps in `.github/workflows/hugo.yaml` after "Build Pagefind index": `npm ci` → `npx playwright install --with-deps chromium` → `npx playwright test`. Gate deploy on it. Pin the Playwright version. Consider a separate job (parallel to LHCI) so it doesn't serialize.
- **ci-local.sh:** add a Playwright step with a loud skip if `npx`/browsers are absent (mirror the existing `need_lhci_dep` preflight pattern).
- **CLAUDE.md:** update the "No npm" line to note the dev-only Playwright exception, and add an E2E row to the testing description. Do this *as part of R5.1* (not before — keep CLAUDE.md accurate to shipped state).
- **.gitignore:** add `test-results/`, `playwright-report/`, `/playwright/.cache/` (node_modules already covered).

## Guardrails / gotchas to carry in

- These tests exist to **catch refactor regressions in R5.2–R5.4** — write them against *current* behavior first (green on today's code), then keep them green through each extraction.
- Real-browser flake: prefer role/text locators + auto-waiting; no arbitrary sleeps.
- The 3 graph runtimes are ~70–80% duplicated (R5.2 target); the graph-mount test is deliberately generic so it survives the extraction.
- Don't let the harness creep into the shipped bundle — it lives entirely under `tests/` + `package.json` devDeps.

## Definition of done for R5.1

Green smoke suite (the 6 above) running locally and in CI over the built site; CLAUDE.md + ci-local.sh updated; deploy gated on it. Then R5.2 opens.
